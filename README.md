# cascade_fir

Исследовательская реализация каскадной формы линейно-фазового КИХ-фильтра,
рассчитанного методом частотной выборки.

## Реализованный алгоритм

`decompose_exact_fs_full()` выполняет следующие операции:

1. Берёт нулевые частотные выборки непосредственно из сохранённого массива
   `DirectFIR::frequency_samples`. Нули не распознаются по округлённым
   коэффициентам `h[n]`.
2. Удаляет известную часть без последовательной дефляции. Для
   `H = H0 H1` и `1 - z^-N = H0 H2` используется тождество
   `H H2 = H1 (1 - z^-N)`; сокращённый палиндромный полином `H1`
   получается из первого блока коэффициентов свёртки `H H2`.
3. Решает оставшуюся задачу непосредственно в базисе Чебышёва. Начальные
   приближения даёт colleague-матрица LAPACK, затем весь набор корней
   одновременно уточняется методом Аберта—Эрлиха с 200 десятичными знаками.
4. Проверяет весь набор корней обратной пересборкой в повышенной точности.
5. Адаптивно объединяет корневые сомножители в FIR-блоки. Для каждого порядка
   сравниваются частотный и time-domain порядок блоков, а компенсационный gain
   перераспределяется по сетке вариантов. Из прошедших импульсную проверку
   реализаций выбирается вариант с минимальным внутренним пиком и стоимостью.
   Сначала проверяется `long double`, затем double-double (~31 десятичный знак),
   34-разрядный `binary128` и резервный 50-разрядный Boost.Multiprecision.

`decompose_exact_fs()` возвращает только аналитическую редукцию с остатком.
Это промежуточное представление для анализа степени, а не рекомендуемый
runtime-каскад. `decompose_exact_fs_structured()` сохранён как совместимый
псевдоним полного пути.

## Офлайн-артефакт каскада

Готовую `CascadeDecomposition` можно один раз сохранить через
`make_cascade_artifact()` и `save_cascade_artifact()`, а в рабочем запуске
загрузить через `load_cascade_artifact()`. Формат содержит версию, точные
битовые представления коэффициентов, спецификацию, выбранную runtime-точность,
диагностику и контрольную сумму. `cascade_artifact_matches()` проверяет, что
артефакт относится именно к переданному `DirectFIR`.

Выбор арифметики хранится в `CascadeDecomposition::runtime_precision` и не
управляется полями диагностики. Загрузчик дополнительно проверяет размеры,
степень полинома, конечность исполняемых коэффициентов, индексы и полноту
порядка звеньев.

## Проверенные результаты

Результаты `exact_fs_diagnostics` для репрезентативных случаев:

| Случай | Известные/извлечённые нули | Степень остатка после аналитического этапа | Максимальный порядок runtime-блока | Максимальная ошибка импульса |
|---|---:|---:|---:|---:|
| N65 | 44/44 | 20 | 4 | `8.27e-15` |
| N97 | 64/64 | 32 | 4 | `3.36e-14` |
| N129 | 86/86 | 42 | 4 | `2.95e-12` |
| N129, ненулевая переходная полоса | 64/64 | 64 | 4 | `1.10e-11` |
| N257, ненулевая переходная полоса | 128/128 | 128 | 4 | `1.23e-9` |
| N513, ненулевая переходная полоса | 256/256 | 256 | 8 | `5.40e-10` |

Для N513 `long double` недостаточен из-за обусловленности короткоблочного
представления. После transient-aware упорядочивания случай получает статус
`HighPrecisionShortBlockCascade` и исполняется в double-double. Переносимый
scalar-kernel использует error-free transforms, а на x86 с AVX2/FMA тот же
артефакт автоматически получает векторный kernel для блоков порядка 4 и 8.
Численная проверка декомпозиции всегда выполняется переносимым scalar-kernel;
AVX2 дополнительно проходит те же impulse/signal допуски. N≤257 остаются на
нативном backend. `binary128` и 50-разрядный Boost.Multiprecision остаются
резервом, если double-double не проходит проверку.

## Сборка и проверка

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j2
ctest --test-dir build --output-on-failure -L core
ctest --test-dir build --output-on-failure -L exact_fs
./build/exact_fs_diagnostics
./build/exact_fs_diagnostics N513T
```

Тесты exact-first проверяют точное число известных нулей, complementary-
тождество, теоретическую степень остатка, чётные и нечётные N вплоть до N513,
полное покрытие корней, ненулевые переходные выборки, выбор точности runtime,
scalar/AVX2 dispatch, гарантированный direct fallback и отсутствие NaN/Inf.

## Benchmark

`cascade_benchmark` раздельно измеряет проектирование, полное разложение,
инициализацию состояний и установившуюся поотсчётную фильтрацию. Перед
измерением выполняется прогрев, для времени на отсчёт выводится медиана пяти
повторов. Прямая и каскадная формы измеряются отдельно; корректность
дополнительно проверяется на одном и том же входном сигнале. Сбор внутреннего
пика измеряется отдельным проходом и не входит в основное runtime-время.

```bash
./build/cascade_benchmark N257T
./build/cascade_benchmark N513T
```

Необязательные второй и третий аргументы задают число измеряемых отсчётов
каскада и число повторов. Четвёртый аргумент задаёт путь артефакта. Если файла
ещё нет, benchmark выполнит декомпозицию и сохранит его; повторный запуск
загрузит готовый каскад, не запуская поиск корней:

```bash
./build/cascade_benchmark N513T 16384 5 /tmp/n513t.cfir
./build/cascade_benchmark N513T 16384 5 /tmp/n513t.cfir
```

Пятый аргумент `auto`, `scalar` или `avx2` позволяет воспроизводимо сравнить
ISA-kernel без повторной декомпозиции:

```bash
./build/cascade_benchmark N513T 16384 5 /tmp/n513t.cfir scalar
./build/cascade_benchmark N513T 16384 5 /tmp/n513t.cfir avx2
```

Показатель `realtime_x` равен отношению измеренной
пропускной способности каскада к частоте дискретизации спецификации. Результат
зависит от процессора и режима сборки и не является частью correctness-тестов.

Референсный Release-замер текущей машины:

| Случай | Построение | Direct | Каскад | `realtime_x` | Макс. ошибка сигнала |
|---|---:|---:|---:|---:|---:|
| N257T | 19,8 с | 4,16 млн отсч./с | 1,60 млн отсч./с | 33,2 | `2,45e-8` |
| N513T | 68,1 с | 2,03 млн отсч./с | 277 тыс. отсч./с | 5,77 | `4,09e-10` |

На одном артефакте N513T scalar double-double показал 104 тыс. отсч./с,
AVX2/FMA --- 275 тыс. отсч./с, то есть ускорение ISA-kernel составило
примерно 2,65 раза без изменения численной точности артефакта.

Контрольный round-trip артефактов на той же машине сохранил ошибки и checksum
runtime без изменений:

| Случай | Размер артефакта | Загрузка | Декомпозиция при повторном запуске |
|---|---:|---:|---:|
| N257T | 8,4 КБ | 0,066 мс | 0 мс |
| N513T | 12,7 КБ | 0,087 мс | 0 мс |

## Зависимости

Используются уже доступные в системе LAPACK и заголовочная
Boost.Multiprecision. Double-double и AVX2/FMA реализованы внутри проекта;
kernel выбирается во время исполнения, поэтому бинарник сохраняет scalar
fallback и не требует глобального `-march=native`. На компиляторах с
`__float128` резервная 34-разрядная ветка использует binary128, иначе —
`cpp_bin_float<34>`. Проект не загружает и не устанавливает пакеты.
